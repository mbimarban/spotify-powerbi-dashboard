2. Star Schema Theory

Why Star Schema?

Transform flat CSV data into dimensional model for better performance and maintainability.

Problem with Flat Data

Beyonc√© appears in 324 rows with identical artist data (name, popularity, followers) = redundancy and slow performance.

Star Schema Benefits

Benefit Explanation
Eliminate Redundancy Each artist stored once (not 324 times)
Faster Calculations No duplicate aggregation
Easier Maintenance Update artist data in one place
Better Scalability Handle millions of rows efficiently
Industry Standard Best practice for BI/Analytics

6. Calculated Columns (Power Query)

Column 1: track_duration_min

Purpose: Convert milliseconds to minutes (more readable)

Steps

  1. Add Column ‚Üí Custom Column
  2. Name: track_duration_min
  3. Formula: [track_duration_ms] / 60000
  4. OK
  5. Change type: Decimal Number

Column 2: track_duration_sec

Purpose: Alternative format in seconds

Steps

  1. Add Column ‚Üí Custom Column
  2. Name: track_duration_sec
  3. Formula: [track_duration_ms] / 1000
  4. OK
  5. Change type: Decimal Number

Column 3: is_explicit

Purpose: Convert boolean to numeric (for math operations)

Steps

  1. Add Column ‚Üí Custom Column
  2. Name: is_explicit
  3. Formula: if [explicit] = true then 1 else 0
  4. OK
  5. Change type: Whole Number

üí° Power Query vs DAX Decision

Power Query: Simple transformations, one-time calculations at refresh

DAX: Aggregations, context-dependent calculations, relationships

12. DAX Measures & Columns

Create KPI Table

Home ‚Üí Enter Data ‚Üí Create table named "KPI" with formula:

KPI = {0}

Basic Metrics (Display Folder: "1. Basic Metrics")

Total Tracks

Total Tracks = COUNTROWS(FACT_Tracks)

Format: Whole number, 1000 separator

Total Artists

Total Artists = DISTINCTCOUNT(FACT_Tracks[artist_name])

Total Albums

Total Albums = DISTINCTCOUNT(FACT_Tracks[album_id])

Avg Track Popularity

Avg Track Popularity = AVERAGE(FACT_Tracks[track_popularity])

Format: Decimal, 1 decimal place

Avg Track Duration

Avg Track Duration = AVERAGE(FACT_Tracks[track_duration_min])

Format: Decimal, 2 decimal places

Calculated Columns

Explicit Label (FACT_Tracks)

Created in Power Query:

// Power Query M
if [explicit] = true then "Explicit"
else if [explicit] = false then "Clean"
else "Unknown"

Duration Group (FACT_Tracks)

Created in Power Query:

// Power Query M - Conditional Column
if [track_duration_min] < 2 then "0-2 min"
else if [track_duration_min] < 3 then "2-3 min"
else if [track_duration_min] < 4 then "3-4 min"
else if [track_duration_min] < 5 then "4-5 min"
else if [track_duration_min] < 6 then "5-6 min"
else "6+ min"

Decade (DIM_Albums)

Created in Power BI (DAX):

// Right-click DIM_Albums ‚Üí New column
Decade = 
INT(YEAR(DIM_Albums[album_release_date]) / 10) * 10 & "s"

Explanation: Converts year to decade (2023 ‚Üí "2020s", 1967 ‚Üí "1960s")

19. Troubleshooting

Common Issues

Issue: Token Literal Expected (Power Query)

Cause: Starting formula with "=" or incorrect syntax

Solution: Remove leading "=", use lowercase true/false, brackets around column names

Issue: Column Cannot Be Determined (DAX)

Cause: Using table[column] in calculated column context

Solution: Use [column] without table name in calculated columns

Issue: Filters Panel Not Displaying

Cause: UI bug or theme conflict

Solution:

  1. View ‚Üí Filters (toggle off/on)
  2. Restart Power BI Desktop
  3. Use Slicers instead (visual filters)

Issue: Hierarchies Not Showing Year

Cause: Column not recognized as date type

Solution: Change column type to Date in Power Query

Issue: Relationships Not Auto-Detected

Cause: Column names don't match or data types different

Solution: Manually create in Model view, verify column types match

Issue: Column Name With Spaces Won't Work

Example: [track duration min] returns error

Solution: In DAX, use table name: 'FACT_Tracks'[track duration min]

20. Glossary

Data Modeling Terms

Term Definition
Star Schema Data model with central FACT table surrounded by DIM tables
FACT Table Contains metrics/transactions (tracks, sales, events)
DIM Table Contains attributes/descriptors (artists, products, locations)
Primary Key Unique identifier in a table (track_id, artist_name)
Foreign Key Reference to primary key in another table
Cardinality Relationship type (One-to-Many, Many-to-One)
Group By Aggregate rows with same value (deduplication)

Power BI Terms

Term Definition
Power Query ETL tool for data transformation (M language)
DAX Data Analysis Expressions - formula language
Measure Calculated value (Total Tracks, Average Popularity)
Calculated Column New column based on existing data
Slicer Visual filter for user interaction
M Language Power Query formula language

Music Industry Terms

Term Definition
Popularity Spotify's 0-100 score based on recent plays
Explicit Content Music with profanity, violence, or adult themes
Album Type album (full release), single, compilation
Track Individual song
